## 3. 架构模式

采用 **Provider + Repository** 模式：

```
UI层 (Screens/Widgets)
    ↕
状态管理层 (Providers)
    ↕
数据访问层 (DAOs)
    ↕
数据层 (SQLite Database)
```

- **UI层**: 负责展示数据和接收用户交互
- **状态管理层**: 使用Provider管理应用状态，连接UI和数据层
- **数据访问层**: 封装数据库CRUD操作
- **数据层**: SQLite本地数据库

## 4. 核心流程

### 4.1 任务打卡流程
1. 用户在首页查看今日任务列表
2. 点击任务卡片上的"打卡"按钮
3. 系统记录打卡时间和日期
4. 更新累计打卡次数（等级经验值）
5. 更新连续打卡天数（段位经验值）
6. 检查是否触发等级/段位升级

### 4.2 等级计算逻辑
- 等级基于**累计打卡次数**
- 每次打卡累加1次
- 等级不会降级

### 4.3 段位计算逻辑
- 段位基于**当前连续打卡天数**
- 每天至少完成1个任务打卡算连续
- 如果某天没有任何打卡，连续天数清零
- 段位可能因中断而降级

### 4.4 通知提醒流程
1. 用户为任务设置提醒时间
2. 系统注册定时通知
3. 到达时间时推送系统通知
4. 用户点击通知进入App打卡

### 4.5 专注模式流程
1. 用户开启专注模式
2. 系统开始计时
3. 后台监测App使用情况
4. 检测到非白名单App时发送通知提醒
5. 用户结束专注，显示专注时长统计
